## Структура решения

### Новые классы в папке `security/`

1. **SessionStore.java**
   - Хранилище активных сеансов в памяти
   - Использует ConcurrentHashMap для потокобезопасности
   - Автоматически удаляет истекшие сеансы
   - Время жизни сеанса: 24 часа

2. **CookieSessionManager.java**
   - Менеджер для управления сеансами
   - Создание, валидация и инвалидация сеансов
   - Имя cookie: `session_id`
   - Max-Age: 86400 секунд (24 часа)

3. **AuthMiddleware.java**
   - Middleware для проверки авторизации
   - Публичные пути (без авторизации): `/auth/login`, `/auth/signup`, `/health`
   - Возвращает 401 Unauthorized для защищенных ресурсов без валидного сеанса

### Изменения в существующих классах

1. **AuthService.java**
   - Добавлены методы `loginWithCookie()` и `signupWithCookie()`
   - Возвращают UUID пользователя вместо JWT токена
   - Старые методы `login()` и `signup()` сохранены для совместимости

2. **AuthHandler.java**
   - Полностью переработан для использования cookies
   - В ответах устанавливаются HTTP cookies вместо токенов
   - Поддержка logout с удалением cookies

3. **Main.java**
   - Добавлена конфигурация для приложения

## API Endpoints

### POST /auth/signup
Регистрация нового пользователя
```json
{
  "username": "username",
  "password": "password",
  "firstname": "Имя",
  "lastname": "Фамилия"
}
```

### POST /auth/login
Вход в систему
```json
{
  "username": "username",
  "password": "password"
}
```

### POST /auth/logout
Выход из системы (требует валидной cookie)

### GET /api/* (все остальные маршруты)
Автоматически требуют валидной сессии через cookie

## Безопасность

- Пароли хешируются с BCrypt
- Сеансы хранятся на сервере в памяти
- Cookie автоматически удаляется при logout
- Доступ к защищенным ресурсам проверяется на каждый запрос
